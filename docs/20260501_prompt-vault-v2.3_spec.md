## 1. 概要

- 何を作るか: カタログの閲覧強化（全画像フラット表示・タグフィルタ・ソート・横断スライドショー）と設定の拡充（画面方向・画像品質・使用容量表示）
- なぜ作るか: 画像ビューアとしての実用性を高め、プロンプトの境界を意識せず画像を連続閲覧できるようにする
- 誰が使うか: ショウゴ（個人利用）。Pixel 10モバイルChrome + PC Chrome

## 3. 技術選定

| 技術 | 理由 |
|---|---|
| IndexedDB `getAll('images')` | 全画像のフラット表示に必要。既存のdb.jsで対応済み |
| Intersection Observer | カタログの遅延読み込み。大量画像時のDOM負荷軽減。外部ライブラリ不要 |
| `navigator.storage.estimate()` | 使用容量の取得。PWA環境で利用可能 |
| localStorage | 設定永続化。既存のテーマ保存と同じ方式 |

## 4. 機能仕様

### §1: カタログ全画像フラット表示

- 何を: グローバルカタログで全プロンプトの全画像をフラットなグリッドで表示する
- どのように:
  - カタログモードshow()時に、dbGetAll('images')で全画像を取得する
  - 各画像にはpromptIdが紐づいているので、prompts配列と突合して親プロンプトのメタデータ（タイトル・タグ・createdAt）を参照できる状態にする
  - 表示はフル画像(800px)のdata URLをsrcに設定する。サムネイル専用データは持たないため、CSS側でobject-fit: coverとサイズ制約で対処する
  - Intersection Observerで画面内に入ったセルだけimgのsrcを設定する遅延読み込みを実装する
  - セルのアスペクト比は1:1（現行と同じ）
- 制約:
  - 画像が0件の場合は現行と同じ空メッセージを表示
  - 読み込み中はセルの背景色(var(--bg-input))で代替

### §2: カタログのソート

- 何を: カタログ上部にソートUIを設置し、表示順を切り替える
- どのように:
  - ソート基準: 作成日（デフォルト）/ 更新日 / タイトル（親プロンプトの属性で判定）
  - 昇順・降順の切替ボタン
  - UIはカタログビュー上部にツールバーを配置。selectタグ1個 + 昇降ボタン1個
  - ソート状態はセッション内のみ保持（localStorage不要）
  - ソート対象は§4のタグフィルタ適用後の配列
- 制約:
  - 同一プロンプトの画像間の順序はorder値を維持する（ソートはプロンプト単位のグループ順を変える）

### §3: プロンプト横断スライドショー

- 何を: カタログから画像をタップした際、カタログに表示中の全画像を連続スライドショーできるようにする
- どのように:
  - viewer.jsに新しいエントリポイント `openBatch(images, startIndex)` を追加する
    - images: { data, promptId, order } の配列（カタログ側で構築済み）
    - startIndex: タップした画像のインデックス
  - 既存の `open(promptId)` は変更しない。プロンプトカード→ビューアの経路はそのまま維持
  - カタログ側から `pv:openViewerBatch` カスタムイベントを発行し、viewer.jsがリッスンする
  - ビューア内部では viewerImages にbatch配列を入れるだけで、スライド操作・スワイプ・キーボードは既存ロジックがそのまま動く
  - ビューアのタイトル表示は、現在表示中の画像のpromptIdから親プロンプトのtitleを引いて動的に更新する
- 制約:
  - batch配列は§2のソートと§4のフィルタが適用された状態でビューアに渡す
  - images配列のdataフィールドはIndexedDBからの取得済みdata URL。ビューア側で再取得はしない

### §4: タグフィルタリング

- 何を: カタログ上部にタグフィルタを設置し、選択タグを持つプロンプトの画像だけに絞り込む
- どのように:
  - 全プロンプトからタグ一覧を抽出（prompts.jsのallTagsと同ロジック）
  - タグボタンをツールバーに横並び表示（現行プロンプトモードのtag-filterと同じUI）
  - タグ選択→prompts配列をフィルタ→該当promptIdの画像のみ表示
  - 複数タグ同時選択はAND条件とする（選択したタグをすべて持つプロンプトの画像）
  - フィルタ済みの画像配列がそのまま§3のbatchに渡る
  - タグ未選択時は全画像表示（§1の状態）
- 制約:
  - タグはプロンプト単位で付与されている。画像個別のタグは存在しない

### §5: ビューア四象限タップ操作

- 何を: スライドショー表示中、画像表示エリアを四象限に分割し、各領域のタップに操作を割り当てる
- どのように:
  - slideWrapのclickイベントで、クリック位置のX/Y座標をslideWrapの幅・高さの50%を基準に四象限判定する
  - 割り当て:
    - 左下（X < 50%, Y >= 50%）: 前の画像へ move(-1)
    - 右下（X >= 50%, Y >= 50%）: 次の画像へ move(1)
    - 右上（X >= 50%, Y < 50%）: カタログモードに切替 setMode('catalog')
    - 左上（X < 50%, Y < 50%）: ビューアを閉じる close()
  - 現行のslideWrapクリック→close()の挙動を廃止する（四象限操作と競合するため）
  - slideImgのstopPropagationを除去し、slideWrap内のどこをタップしても四象限判定を行う
  - 端（最初の画像で左下タップ、最後の画像で右下タップ）では何も起きない（現行のdisabledと同じ）
  - 上部のボタン群（閉じる・スライド/カタログ切替）は従来通り残す。四象限タップはショートカットとして共存する
- 制約:
  - スワイプ操作との共存: タッチ操作ではtouchstart/touchendのスワイプが優先される。タップ（移動距離50px未満）はclick判定に入る。ブラウザのデフォルト動作で自然に共存する
  - PC環境ではマウスクリックの四象限判定として機能する

### §6: 画面方向設定

- 何を: 設定モーダルに画面方向の選択肢を追加する
- どのように:
  - 3つの選択肢: 「常時縦固定」/「ビューア時のみ横許可」（デフォルト・現行動作）/「常時自由回転」
  - 「常時縦固定」: ビューアを開いても screen.orientation.unlock() を呼ばない
  - 「ビューア時のみ横許可」: 現行の動作（ビューア開→unlock、閉→lock portrait）
  - 「常時自由回転」: 起動時に screen.orientation.unlock() を一度呼び、以後ロックしない
  - viewer.jsのopen/closeでsettings値を参照して分岐
  - UIはsettingsモーダル内にラジオボタン風のボタン群（テーマ選択と同じ見た目）
- 制約:
  - Screen Orientation APIは一部ブラウザで非対応。try-catchで囲む（現行と同じ）

### §7: 画像品質設定

- 何を: 新規追加する画像のリサイズ上限とJPEG品質を設定から変更できるようにする
- どのように:
  - リサイズ上限: 800px（デフォルト）/ 1200px / 元サイズ維持
  - JPEG品質: 75%（デフォルト）/ 85% / 95%
  - UIはsettingsモーダル内にセレクトまたはボタン群
  - prompts.jsのhandleAddImagesが呼ぶresizeImage()の引数を、state.jsのgetSettings()から取得した値で動的に設定する
  - 「元サイズ維持」選択時はresizeImageの最大幅/高さを99999に設定（実質無制限）
  - 既存画像のリサイズは行わない。設定変更後に新規追加された画像から適用
  - サムネイル(400px/JPEG 60%)の品質は固定。変更しない
- 制約:
  - 「元サイズ維持 + 品質95%」はIndexedDB容量を急速に消費する。設定変更時にtoastで注意を表示する

### §8: 使用容量表示

- 何を: 設定モーダルに現在のIndexedDB使用容量を表示する
- どのように:
  - `navigator.storage.estimate()` で usage / quota を取得
  - 表示形式: 「使用中: XX MB / YY MB (ZZ%)」
  - settingsモーダルを開くたびに再取得して表示を更新
  - UIはsettingsモーダルの入出力セクションの上に「ストレージ」セクションとして配置
- 制約:
  - `navigator.storage.estimate()` はブラウザ全体のストレージ使用量を返す（IndexedDB単体ではない）。表示ラベルは「ブラウザストレージ使用量」とし、厳密にIndexedDBのみの値ではないことを示す
  - 非対応ブラウザでは「計測不可」と表示する

### §9: 設定の永続化

- 何を: §6, §7の設定値をlocalStorageに保存し、アプリ再起動後も維持する
- どのように:
  - state.jsに `getSettings()` / `updateSettings(partial)` を追加
  - localStorageキー: `pv-settings`
  - 保存形式: JSON文字列
  - デフォルト値:
    ```json
    {
      "orientation": "viewer-only",
      "imageMaxSize": 800,
      "imageQuality": 0.75
    }
    ```
  - app.jsの初期化時にlocalStorageから読み込み、state.jsにセットする
  - 既存の `pv-theme` は変更しない（後方互換）
- 制約:
  - 設定値が壊れていた場合はデフォルト値にフォールバックする

## 5. テスト方針

| テスト対象 | 方法 | 合格条件 |
|---|---|---|
| §1 全画像表示 | 複数プロンプト（各3枚以上）登録後にカタログモード表示 | 全画像がフラットに表示され、枚数が一致する |
| §2 ソート | 作成日・更新日・タイトル各基準で昇順/降順を切替 | 表示順が正しく変わる |
| §3 横断スライドショー | カタログから画像タップ→スライドで前後に送る | プロンプトの境界を超えて連続送りできる。タイトル表示が追従する |
| §4 タグフィルタ | タグ選択→カタログ絞り込み→スライドショー | 選択タグの画像のみ表示される。スライドショーもフィルタ範囲内で動作 |
| §5 四象限タップ | スライドショーで各象限をタップ | 左下→前、右下→次、右上→カタログ、左上→閉じる。端では送り無反応 |
| §6 画面方向 | 3モード各々で設定→ビューア開閉 | 設定に応じた画面方向制御が動作する |
| §7 画像品質 | 各設定で画像追加→IndexedDB内のデータサイズを確認 | 設定値に応じたリサイズ・品質で保存される |
| §8 容量表示 | 設定モーダルを開く | 使用量が数値で表示される |
| §9 永続化 | 設定変更→アプリ再起動 | 設定値が維持されている |
