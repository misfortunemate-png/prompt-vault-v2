## 作業範囲

- 何を: カタログ閲覧強化（全画像フラット表示・タグフィルタ・ソート・横断スライドショー・四象限タップ）と設定拡充（画面方向・画像品質・使用容量表示・永続化）
- なぜ: 仕様書 §1〜§9
- どこで: 既存ファイルへの改修のみ。対象は index.html, js/catalog.js, js/viewer.js, js/settings.js, js/state.js, js/utils.js, js/app.js, sw.js, README.md

## 参照ドキュメント

- 仕様書: 20260501_prompt-vault-v2.3_spec.md（本指示書と同じディレクトリ）
- CLAUDE.md: リポジトリルートに配置済み。作業開始前に必ず読むこと
- _STATUS.md: 作業開始前に必ず読むこと。中断時は更新すること

## 実装順序（仕様書§6より）

1. §9（設定永続化基盤）→ §6（画面方向）→ §7（画像品質）→ §8（容量表示）
2. §1（全画像フラット表示）→ §4（タグフィルタ）→ §2（ソート）
3. §3（横断スライドショー）→ §5（四象限タップ）

各§を完了するごとにコミットすること。

## 禁止事項

- 仕様書に記載のない機能を追加しない
- 仕様書に記載のないファイルを新規作成しない
- 仕様書の原本を改変しない
- 仕様外の設計判断が必要な場合は作業を停止し報告する
- 既存のモジュール分割構造（state.jsを介した疎結合）を崩さない
- db.jsのスキーマ（IndexedDBのobjectStore構成）を変更しない

## 完了条件

- 仕様書§1〜§9の全機能が実装されていること
- 仕様書§5テスト方針の全項目が合格していること
- sw.jsのキャッシュバージョンが `pv-v2.3` に更新されていること
- README.mdの変更履歴・機能一覧が更新されていること
- _STATUS.mdが最新の状態を反映していること
- `git log --oneline` で§ごとのコミットが確認できること

## 注意事項

- index.htmlは615行ある。CSSとHTMLが一体のシェルなので、変更箇所を最小限にすること
- viewer.jsの既存エントリポイント open(promptId) は変更しない。openBatch を追加する形で拡張する
- catalog.jsは現在30行と小さい。全画像取得・フィルタ・ソートの追加で大幅に拡張されるが、state.jsとの疎結合を維持すること
- settings.jsのapplyTheme()は触らない。新しい設定は別セクションとして追加する
